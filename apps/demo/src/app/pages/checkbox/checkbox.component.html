Checkbox Documentation

This component includes both ip-checkbox and ip-checkbox-list; which one to use depends on your needs:

  • ip-checkbox: For a simple checkbox
  • ip-checkbox-list: For a list of checkboxes

ip-checkbox

Description

IpCheckbox is a Stencil.js component that displays a single checkbox.

Usage

To use this component, you must add the properties as attributes.

Component Tag:

Use the <ip-checkbox> tag in your JSX/HTML.

Properties

Property Attribute Required Description Type Default
id id Yes The ID of the input for accessibility string undefined
name name Yes The name of the input for accessibility string undefined
defaultChecked default-checked No Sets the checkbox to checked by default boolean false
disabled disabled No Disables the checkbox boolean false

Example:

<ip-checkbox default-checked="true" disabled="true" id="Male"></ip-checkbox>

Content

There is a predefined label slot for the label before the checkbox, which will be the content inside the tag.

Example:

<ip-checkbox default-checked="true" disabled="true" id="checked"> Checked ! </ip-checkbox>

Customization:

We have a set of predefined variables used for customizing the checkbox:

  • --primary-color
  • --secondary-color
  • --font-size
  • --checkbox-size

To update the values, use the following CSS:

ip-checkbox {
  --primary-color: #006342;
  --secondary-color: #000000;
}

Listening to Events

To listen to events emitted by the ip-checkbox component, open your browser's console. Event details will be displayed in the console when the checkbox state changes.

Example JavaScript Code:

const checkbox = document.querySelector('ip-checkbox');

checkbox.addEventListener('change', event => {
  const isChecked = event.detail.checked;
  console.log(isChecked);
});

ip-checkbox-list

Usage

To use this component, you need to add the properties as attributes.

Component Tag:

Use the <ip-checkbox-list> tag in your JSX/HTML.

Properties

Property Attribute Required Description Type Default
options options Yes An array of objects used to pass options as checkboxes string undefined
id id Yes ID of the input for accessibility string undefined
defaultChecked default-checked No When you need to set the checkbox to checked by default boolean false
disabled disabled No When you need to set the checkbox to disabled boolean false
legend legend No The legend for the list of checkboxes string undefined

Example:

<ip-checkbox-list options='[{"id": "option1", "label": "Option 1"}, {"id": "option2", "label": "Option 2"}]'></ip-checkbox-list>

Customization

We have a set of predefined variables used to customize the checkbox:

  • --primary-color
  • --secondary-color
  • --font-size
  • --checkbox-size

To update the values, use the following CSS:

ip-checkbox-list {
  --primary-color: #006342;
  --secondary-color: #000000;
}

Listening to Events

To listen to events emitted by the ip-checkbox and ip-checkbox-list components, open your browser's console. Event details will be displayed in the console when checkboxes are modified.

Example JavaScript Code:

const checkboxList = document.querySelector('ip-checkbox-list');
const checkbox = document.querySelector('ip-checkbox');

checkboxList.addEventListener('change', event => {
  console.log(event.detail);
});

checkbox.addEventListener('change', event => {
  const isChecked = event.detail.checked;
  console.log(isChecked);
});
